Learning lists

Fri 04 October 2019 | tags: Pythonintroductionlists

Disclaimer

What follows is not written to be 100% accurate in all the technical detail. It is written to help people with limited knowledge of computers to get started with using python. As such, I have tried to limit jargon so if you already know terms like "object" and "class", or what a command line is, then this page ain't for you. Feel free to stick around though.

In the last post we talked about assigning information (data) to a variable (a short name) that python can use, and that is often easier to type. So this allows us to take a long piece of data like

"Let each morn be better than its eve and each morrow richer than its yesterday."

and store it in a variable

morn = 'Let each morn be better than its eve and each morrow richer than its yesterday.'

Whats cool about this is it allows us to call and work with that variable rapidly

print(morn)

'Let each morn be better than its eve and each morrow richer than its yesterday.'

That is basically a means of storing data in python, if you have been paying attention you probably suspect that there's more, you're very smart.

Lists

Lists are a cool and simple to store multiple pieces of data in python in a neat way. This is useful for a few reasons.

1. It gives us quick access to variables.
2. It helps to keep our code tidy.
3. It is one of the ways we can automate operations on multiple variables rapidly (we'll cover this later on).
4. It can store variables of different data types, ints, strings, etc.

Making lists in python is really easy; just surround it in square brackets "[]"!

age = 29
name = 'jean'
occupation = 'lecturer'
height = 1.75

[age, name, occupation, height]#this bit is the list

See how easy that was? First we open our square brackets, then we add our variables with each one being seperated by a coma. The coma is important, without it list won't work for you so make sure you are paying attention when you make a list.

Lists are also data

Another cool thing is that a list is also a piece of data, which means a whole list can be assigned to a variable just like a string of int. 

age = 29
name = 'jean'
occupation = 'lecturer'
height = 1.75

l = [age, name, occupation, height]

In the above chunk we have taken all our variables, assigned them to a list and also assigned the list itself to a variable. We can do that because a list is also a datatype in python! Take a second to think about what that means.

OK, you got it?

It means that we can also store a list inside another list! We can make a list... of lists!

age = 29
name = 'jean'
occupation = 'lecturer'


l1 = [age, name, occupation, height]

eye = 'blue/grey'
hair = 'red'
height = 1.75

l2 = [eye, hair, height]


l3 = [l1, l2]#a list of lists

Lists seem simple, and that's mainly because they are. Simple but powerful. I'm aware that thier power is probably not overly clear right now but I promise it shall all become clear when we start to look at things like automating actions. For right now, all you need to take with you is that a list is just what it sounds like, a collection of data packed into a single item. That item can also be assigned to a variable and can then be stored inside a list itself.

Its not a complicated thing but it'll be really useful to you as you progress.

You're doing excelent so far, and the next post will be the last of the datatypes we're going to cover (for a while) and once thats covered we'll look at the first of our python actions, namely indexing.

See you then,
