Variables

You now know that python handles everyday language as strings and that numbers can be treated as either ints or floats, noice! What you've actually done is played with data objects in python for the first time, and you've already done so with 3 different classes of data.

Data is just information, and, in very simplistic terms, coding languages like python are really good at doing stuff with information. Strings, ints, and floats are just different types (or classes) of data. There are lots of other data classes but for now we're just going to stick with these basic one.

The first thing we're going to cover is called assigning variables. This just refers to the process of taking a piece of data (information) and giving it a name that python can use to quickly find and work with that data. This is really easy to do with python.

Lets take a piece of text that we want to work with.

"When a thought of war comes, oppose it by a stronger thought of peace."

You can also think of that sentence as a pice of information/data, or as an object within python. This object happens to be of the string class but that doesn't matter for right now. What matters is that you can take that whole object (everything between the quotation marks) and give it short name that you can use to refer to the entire object without having to type it all out everytime you want to do something with it. This process is called assigning to a variable and it's really really easy.

s = "When a thought of war comes, oppose it by a stronger thought of peace."

The chunk of code above is simple but it packs a punch, lets walk through it.

The lower case s at the begining is the short name (or variable) that we are giving to the string, the = sign is the symbol that we use to do the assignment, and the string is object that we are assigning. To translate this into everyday speech it would read as:

This string is now called s.

One of the things that trips people up is the grammer of the code chunk is acctually the other way around:

s is the name of that string.

The second version is more accurate and I've found it useful to just settle that structure in my mind:

  1. variable name
  2. =
  3. the piece of data

Lets look at another example.

m = 12 
a = 40 
f = m*a

print(f)

In the first line, we assign m the value of 12. In the second line we give a the value of 40 and in the last line we give f the value of m multiplied by a. So in this example we have assigned three objects to three variables:

Object 1. the int 12
Object 2. the int 40
Object 3. the product of 12 multiplied by 40

Now, they are called variables because they can change, or to put it another way, the object assigned to that variable can be updated. Lets imagine that m is the mass of a car and a is its acceleration. Lets say we are getting m and a from some measurement devices that are connected to our computer, like a weighing scales for m and speedometer for a. By multiplying m and a together we get the Force the car is exherting as it travels (f). If the car speeds up or slows down a also changes, likewise if someone exits or enters the car m will change. In both cases the object is changing, or varying, which means that f is also changing, or varying.... get it? They're variables... Sorry.

Naming variables

You could have used pretty much any word or letter as the variable names, excluding spaces and punctuation. This means that 

m = 12 
a = 40 
f = m*a

print(f)

is the same as 

banana = 12 
apple = 40 
fruit = m*a

print(fruit)

However you can use _ (underscores) and you're encouraged to do so.

car_mass = 12 
car_acceleration = 40 
car_force = m*a

print(car_force)

It's best to use simple and informative names and to be consistent in your naming convention. Without going into why, python users tend to prefer using underscores when linking words together and to balance brevity with clarity so that 'car_accel' might be better than 'car_acceleration' because the later is cumbersome to type, although personally, I think both options need work. As you get more practice in you will develop a taste for how you like to name variables.

So now you know how to assign a variable, and you know that strings, ints, and floats are data types that can be assigned.
