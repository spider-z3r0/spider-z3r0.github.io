Variables.

Sun 22 September 2019 | tags: Pythonintroductionvariables

Disclaimer

What follows is not written to be 100% accurate in all the technical detail. It is written to help people with limited knowledge of computers to get started with using python. As such, I have tried to limit jargon so if you already know terms like "object" and "class", or what a command line is, then this page ain't for you. Feel free to stick around though.

Variables

You now know that python handles everyday language as strings and that numbers can be treated as either ints or floats, noice! What you've actually done is played with data in python for the first time, and you've already done so with 3 different classes of data.

Data is just information, and, in very simplistic terms, coding languages like python are really good at doing stuff with information. Strings, ints, and floats are just different types of data and this means that we can perform lots of different operations (which is just a coders way of saying "do lots of things") with them.

The first thing we're going to cover is called assigning variables. This just refers to the process of taking a piece of data (information) and giving it a name that python can use to quickly find and work with that data. This is really easy to do with python.

Lets take a piece of text 

"When a thought of war comes, oppose it by a stronger thought of peace."

that we want to work with. That sentence is a piece of data of the string class, we can assign the whole string a short name to make life easier.

s = "When a thought of war comes, oppose it by a stronger thought of peace."

The chunk of code above is simple but it packs a punch, lets walk through it. The lower case 's' at the begining is the short name (or variable) that we are giving to the string, the '=' sign is the symbol that we use to do the assignment, and the string is... well the string. To translate this into everyday speech it would read as:

This string is now called s.

One of the things that trips people up is the grammer of the code chunk is acctually the other way around:

s is the name of that string.

The second version is more accurate and I've found it useful to just settle that structure in my mind:

1. variable name 
2. =
3. the piece of data

Lets look at another example.

m = 12 
a = 40 
f = m*a

print(f)

In the first line, we give m the value of 12. In the second line we give a the value of 40 and in the last line we give f the value of m multiplied by a. Now, the thing is, we don't need to acctually know what m and a are for this to work. Imagine we had some system that automatically fed data into python, like a digital scale and a speedometer, or a website that let users enter data. Python could recieve the data for m and a and then produce f as many times as we want. The code to do that is more complex than the chunk above, but the chunk above is a step towards getting there.

So now you know how to assign a variable, and you know that strings, ints, and floats are data types that can be assigned. Next time we'll go over more complex types of data objects, and from there we'll get to doing actually work with objects.

Well done so far.
